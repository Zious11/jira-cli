# S-3.06: Codify pre-merge numeric claim checker for BC heading/body drift (DRIFT-001)

## Context

DRIFT-001 (STATE.md drift register) documents a recurring process failure: numeric count claims
in BC headings (e.g., `bc-2-issue-read.md` frontmatter `total_bcs: 91`) drift from the actual
count of `#### BC-` headings in the body. This was the single most-repeated finding class across
adversarial passes P21, P22, P23, P24 — four recurrences. It was escalated to "codify as
self-improvement story" in STATE.md.

The root cause: count-changing edits to a BC file update either the frontmatter field OR the
body headings, but not both. A grep sweep would catch the drift immediately, but no automated
check runs pre-merge.

### The fix

A short shell script `scripts/check-spec-counts.sh` that:
1. For each `bc-*.md` file in `.factory/specs/prd/`, counts actual `#### BC-` headings via grep.
2. Extracts `definitional_count:` from frontmatter via `grep`.
3. Reports mismatches as errors (exit 1).
4. Also checks `total_nfrs:` in `nfr-catalog.md` against actual `^| \*\*NFR-` rows.
5. Also checks `total_holdouts:` in `holdout-scenarios.md` against actual `^### H-` headings.

A lessons codification document `rules/lessons-codification.md` records the DRIFT-001 pattern
and the checker as the mitigation, so future adversarial passes reference it directly.

`tdd_mode: facade` — this is a shell script + documentation story. No Rust code changes.
The script itself is the deliverable; it can be verified by running it against the current spec
corpus and confirming it exits 0 (all counts match as of Pass 28 convergence).

## Acceptance Criteria

**AC-001**: Given `scripts/check-spec-counts.sh` is created and executable, when it is run against
the current `.factory/specs/prd/` directory, then it exits 0 — confirming all `definitional_count`
frontmatter values match the actual `#### BC-` heading count in each bc-*.md file.

**AC-002**: Given a bc file is temporarily edited so that `definitional_count: 49` is changed to
`definitional_count: 50` (but no heading added), when `check-spec-counts.sh` is run, then it
exits 1 and stderr reports the specific file and the mismatch (`found 49, declared 50`).

**AC-003**: Given `check-spec-counts.sh` is created, when it runs against `nfr-catalog.md`, then
it verifies `total_nfrs: 41` matches the actual `^| \*\*NFR-` count. If a future pass adds an NFR
without updating the frontmatter, the script exits 1.

**AC-004**: Given `.factory/rules/lessons-codification.md` is created, when it is opened, then it
contains the DRIFT-001 description, the root cause (frontmatter/body count split), the mitigation

**AC-005**: Given CLAUDE.md is updated, when a reader looks at the "AI Agent Notes" section, then
they find a reference to `scripts/check-spec-counts.sh` and its purpose (pre-merge spec count
verification), so future AI agents know to run it after editing spec files.

## Implementation Notes

### Script outline (`scripts/check-spec-counts.sh`)

```bash
#!/usr/bin/env bash
set -euo pipefail
FACTORY=".factory/specs/prd"
ERRORS=0

# Check each bc-N-*.md file
for f in "$FACTORY"/bc-*.md; do
  actual=$(grep -c '^#### BC-' "$f" || true)
  declared=$(grep '^definitional_count:' "$f" | awk '{print $2}')
  if [ "$actual" != "$declared" ]; then
    echo "ERROR: $f: actual #### BC- count=$actual, frontmatter definitional_count=$declared"
    ERRORS=$((ERRORS+1))
  fi
done

# Check nfr-catalog.md
nfr_actual=$(grep -c '^| \*\*NFR-' "$FACTORY/nfr-catalog.md" || true)
nfr_declared=$(grep '^total_nfrs:' "$FACTORY/nfr-catalog.md" | awk '{print $2}')
if [ "$nfr_actual" != "$nfr_declared" ]; then
  echo "ERROR: nfr-catalog.md: actual NFR count=$nfr_actual, frontmatter total_nfrs=$nfr_declared"
  ERRORS=$((ERRORS+1))
fi

# Check holdout-scenarios.md
h_actual=$(grep -c '^### H-' "$FACTORY/holdout-scenarios.md" || true)
h_declared=$(grep '^total_holdouts:' "$FACTORY/holdout-scenarios.md" | awk '{print $2}')
if [ "$h_actual" != "$h_declared" ]; then
  echo "ERROR: holdout-scenarios.md: actual H- count=$h_actual, frontmatter total_holdouts=$h_declared"
  ERRORS=$((ERRORS+1))
fi

if [ "$ERRORS" -gt 0 ]; then
  echo "FAIL: $ERRORS spec count mismatch(es). Fix frontmatter or body before merging."
  exit 1
fi
echo "OK: all spec counts verified."
```

## Architecture Compliance Rules

- Script lives in `scripts/` (sibling to source code, not inside `.factory/`).
- Script uses only POSIX-portable shell constructs (`grep`, `awk`, `bash` shebang for safety).
  No Python, no jq, no external tools not available in standard macOS/Linux environments.
- The script does NOT auto-fix mismatches — it reports only. Auto-fix could mask the root cause.
- Reference: `CANONICAL-COUNTS.md` for the counts being verified.

## Risk / Dependencies

- **Risk: frontmatter grep fragility.** `grep '^total_nfrs:'` may not handle YAML with spaces
  differently from the pattern. Verify against actual frontmatter format before using `awk '{print $2}'`.
  The bc-*.md files use `definitional_count: 46` format (space-separated); this is safe for awk.
- **Risk: script path.** If the script is run from the wrong directory (not the repo root), the
  `FACTORY` relative path fails. Add a check: `[ -d ".factory" ] || { echo "Run from repo root"; exit 1; }`.
- **Low complexity, low risk.** This is the simplest story in Wave 3.
