# S-3.09: Formally record PKCE deferral (SD-001 → DEFER; ADR-0013)

## Context

NFR-S-A (PKCE enforcement) was identified during Phase 1 as a security hardening requirement.
SD-001 (Security Decision 001) captured the investigation into PKCE adoption. ADR-0013 formally
records the deferral decision: PKCE adoption requires Atlassian to support PKCE for the OAuth
2.0 authorization code flow in their developer platform. As of the Phase 2 cutoff (2026-05-06),
Atlassian's OAuth 2.0 implementation does not support PKCE for the authorization code flow used
by `jr`. The embedded `jr` OAuth app (ADR-0006) further constrains this — the callback URL and
flow are fixed, and adding PKCE mid-flow would require Atlassian platform changes.

Wave-Plan.md lists NFR-S-A as a Wave 3 story. This is that story. It is documentation-only:
no code changes, no new tests. The deliverable is an updated NFR catalog routing column and a
cross-reference comment trail that makes the deferral discoverable and reversible.

**Reactivation trigger**: When Atlassian announces PKCE support for OAuth 2.0 authorization code
flow (tracked under ADR-0013 section "Reactivation"), this story's closure should be reopened
and a new S-4.NN (or equivalent) created for the implementation.

**Risk context**: R-M1 (MEDIUM — user token security) is partially mitigated by the embedded
OAuth app (ADR-0006) which uses short-lived tokens and the fixed callback URL. PKCE would
further reduce CSRF risk but cannot be implemented client-side alone.

## Risk / Dependencies

- **No code risk.** Documentation-only changes.
- **R-M1 partial mitigation**: The embedded OAuth app (ADR-0006) mitigates some CSRF risk via
  fixed callback URL + short-lived tokens. PKCE would provide additional mitigation but is not
  currently achievable without Atlassian platform changes.
- **Reactivation risk**: If Atlassian silently adds PKCE support without announcement, the
  deferral may be stale. The Reactivation section in ADR-0013 documents the monitoring action.

## Out of Scope

- PKCE implementation — requires Atlassian platform PKCE support (not yet available).
- Changes to the embedded OAuth flow (ADR-0006) — scope freeze for v0.5.
- Other SD-* closures — see S-3.08 for DOCUMENT-AS-IS LOW NFR closures.
