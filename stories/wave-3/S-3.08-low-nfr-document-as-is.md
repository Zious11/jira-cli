## Implementation Notes

### Source comment format convention

All comments follow this format:
```rust
// NFR-O-I: ADF node types mention/emoji/inlineCard/media fall through to `_` here.
// This is deliberate per issue #202. To improve in v2: render @<displayName> for mentions,
// :emoji: for emojis, [<title>](url) for inlineCard. Not prioritized for v0.5.
```

The `// NFR-<ID>:` prefix makes these comments grep-able by NFR ID.
