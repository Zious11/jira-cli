# S-1.01: Pin all GitHub Actions to full commit SHAs

## Context

All GitHub Actions references in `ci.yml` and `release.yml` use mutable version tags
(`actions/checkout@v6`, `Swatinem/rust-cache@v2`, `EmbarkStudios/cargo-deny-action@v2`,
`softprops/action-gh-release@v2`, `actions/upload-artifact@v7`,
`actions/download-artifact@v8`, `codecov/codecov-action@v6`,
`taiki-e/install-action@cargo-llvm-cov`, `dtolnay/rust-toolchain@stable`,
`dtolnay/rust-toolchain@1.85.0`). A tag can be force-pushed to point at
malicious content at any time.

This is a HIGH severity supply-chain risk (NFR-S-E, cicd-setup.md GAP-1,
risk-register R-H6) because `release.yml` injects `JR_BUILD_OAUTH_CLIENT_ID` and
`JR_BUILD_OAUTH_CLIENT_SECRET` from repository secrets. A compromised action
tag in the release pipeline has direct read access to these secrets in the runner
environment.

The fix is purely mechanical: replace every `@<tag>` with `@<sha256>  # <tag>`
using the canonical SHA for each action at the time of implementation. Dependabot
(already configured for the `github-actions` ecosystem) will then keep SHAs current
via automated PRs.

## Behavioral Contract

No BC anchor exists for CI infrastructure changes. This story is anchored entirely to
NFR-S-E (HIGH) from the NFR catalog. The AC below defines the target state.

## Acceptance Criteria

**AC-001** (traces to NFR-S-E, cicd-setup.md GAP-1): Every `uses:` line in
`.github/workflows/ci.yml` and `.github/workflows/release.yml` ends in a full 40-hex-character
commit SHA, not a version tag. Version tag is preserved as a trailing comment for human
readability (e.g., `uses: actions/checkout@11bd71901bbe5b1630ceea73d27597364c9af683  # v4.2.2`).

**AC-002** (traces to NFR-S-E): No `uses:` line retains a floating `@v<N>`, `@stable`,
`@<semver>`, or `@<tool-name>` reference. The presence of any such reference causes CI to fail.

**AC-003** (traces to NFR-S-E): The pinned SHAs correspond to the same action versions
currently in use (not a downgrade or upgrade). Reviewer verifies SHA → tag mapping via
`https://github.com/<org>/<action>/tags` or `gh api`.

**AC-004** (traces to NFR-S-E): Dependabot `.github/dependabot.yml` already covers the
`github-actions` ecosystem with weekly updates. No change to `dependabot.yml` is required
(it was already configured to watch GitHub Actions); confirm it is still active after this
story merges.

## Implementation Notes

### Actions to pin (ci.yml)

| Action | Tag currently used | Suggested approach |
|--------|-------------------|-------------------|
| `actions/checkout` | `@v6` | Pin to SHA for v4.2.2 (latest stable) |
| `Swatinem/rust-cache` | `@v2` | Pin to SHA for v2.x latest |
| `dtolnay/rust-toolchain` | `@stable`, `@1.85.0` | Pin stable and 1.85.0 variant SHAs |
| `taiki-e/install-action` | `@cargo-llvm-cov` | Pin to SHA |
| `EmbarkStudios/cargo-deny-action` | `@v2` | Pin to SHA |
| `codecov/codecov-action` | `@v6` | Pin to SHA |

### Actions to pin (release.yml)

| Action | Tag currently used | Risk level |
|--------|-------------------|-----------|
| `actions/checkout` | `@v6` | High — runs in all jobs |
| `actions/upload-artifact` | `@v7` | Medium |
| `actions/download-artifact` | `@v8` | Medium |
| `softprops/action-gh-release` | `@v2` | HIGH — `contents: write` + OAuth secrets in env |

### Tooling tip

Use `pin-github-action` (GitHub CLI extension) or look up each SHA from the action's
tags page. Format: `uses: <owner>/<action>@<40-hex-sha>  # <tag>`.

Highest-risk action: `softprops/action-gh-release@v2` — runs with `contents: write`
and receives OAuth client credentials. Pin this one first.

## Test Plan

This story has no automated tests (pure CI configuration change). Verification is manual:

1. **Pre-implementation audit**: Run `grep -rn '@v' .github/workflows/` to inventory all
   floating tags.
2. **Post-implementation audit**: Run `grep -E 'uses:.*@[a-f0-9]{40}' .github/workflows/ | wc -l`
   and confirm count equals total number of `uses:` lines.
3. **CI run**: Push to a branch and verify all jobs pass without timeout or action resolution errors.
4. **Dependabot check**: Confirm `dependabot.yml` `package-ecosystem: github-actions` is intact.

`tdd_mode: facade` — there is no meaningful unit-under-test here; the quality gate
is CI passing with all SHAs resolving correctly.

## Architecture Compliance Rules

- Only `.github/workflows/` files are modified. No source code changes.

## Risk / Dependencies

- **No Wave 0 dependency.** This is independent infrastructure work.
- **Risk: SHA staleness.** SHAs pinned at implementation time will drift from latest. Dependabot
  mitigates by opening weekly update PRs. Accept this as normal CI maintenance overhead.
- **Risk: SHA lookup error.** If wrong SHA used (e.g., typo), CI will fail with "unable to
  resolve action". Mitigate by verifying SHA → tag in browser before committing.
- **No breaking change.** Same action code runs; only the resolution mechanism changes from
  mutable tag to immutable SHA.
